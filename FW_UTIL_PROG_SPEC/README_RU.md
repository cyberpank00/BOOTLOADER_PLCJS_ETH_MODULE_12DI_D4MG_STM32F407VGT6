# Спецификация графической утилиты прошивки

## Кратко

Нужно создать графическую утилиту для прошивки Ethernet-модуля на STM32F407.
Утилита должна использовать существующий Modbus TCP OTA-процесс из
`tools/fw_update.py` и заменить консольный сценарий удобным интерфейсом для
оператора.

## Цели

- Читать статус бутлоадера
- Выбирать файл прошивки `.bin`
- Загружать прошивку по Modbus TCP
- Показывать прогресс и ошибки
- Поддерживать `abort` и `reboot`

## Что не входит в задачу

- Изменение протокола бутлоадера
- Замена текущего OTA-формата
- Добавление нового транспорта прошивки

## Функциональные требования

### Подключение

- Подключение по IP-адресу и TCP-порту
- Использование того же набора регистров и команд, что и в `fw_update.py`
- Возможность повторить подключение при ошибке

### Просмотр статуса

Показывать:

- состояние бутлоадера
- последнюю ошибку
- признак валидности приложения
- версию приложения
- product ID
- hardware revision
- количество полученных блоков / общее число блоков
- размер образа
- CRC32 образа
- статус команды
- признак валидного staging

### Прошивка

- Выбор локального файла `.bin`
- Расчёт и отображение размера образа и CRC32
- Отправка `BEGIN_UPDATE`
- Передача блоков с отображением прогресса
- Отправка `FINALIZE_UPDATE`
- Отправка `INSTALL_UPDATE`
- Понятный вывод результата и ошибок

### Команды управления

- `Status`
- `Abort`
- `Reboot`
- `Select firmware`
- `Flash`

## Требования к интерфейсу

- Одно окно
- Поля IP-адреса и порта
- Кнопка выбора файла прошивки
- Индикатор прогресса передачи
- Окно логов
- Панель статуса бутлоадера

## Обработка ошибок

- Показ сетевых ошибок
- Перевод кодов ошибок бутлоадера в человекочитаемый вид
- Отсутствие зависания интерфейса во время прошивки
- Сохранение логов текущей сессии

## Критерии приёмки

- Оператор может читать статус бутлоадера без консоли
- Оператор может успешно прошить корректный `.bin`
- Утилита показывает прогресс передачи
- Утилита корректно сообщает об ошибке CRC и обрыве передачи
- Утилита умеет выполнять `abort`
- Утилита умеет выполнять `reboot`

## Основа реализации

Графическая утилита должна быть основана на логике:

- `tools/fw_update.py`
- текущей карте регистров бутлоадера
- протестированном сценарии warm reset между app и bootloader
